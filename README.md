# CS-330-Comp-Graphic-and-Visualization
Working on this 3D scene project helped me better understand both the design and development sides of building software. When   I approach designing software, I usually start by thinking about what I want the final result to look like and how everything should function from a user's perspective. For this project, I focused on creating a simple desk scene and then broke it down into smaller parts like the table, monitor, and lamp. This made the design procress feel more manageable instead of overwhelming. 
  One of the biggest design skills I developed was learning how to think in 3D space. I had to work with scaling, positioning, and rotating objects so they looked realistic together. I also learned how lighting and tectures can completely change how a scene looks, which added another layer to the design process. Throughout the project, I followed a step by step approach where I created basic shapes first, adjusted their placement, and then added textures and lighting. Breaking things into smaller steps is something I can use in my future projects because it kept me organized and helped me stay on track.
  When it comes to developing programs, I tend to start simple and build up from there. This project reinforced that approach because trying to do everything at once would have been too difficult. One strategy I used was testing my program frequently after small changes so I could catch mistakes early. I also relied a lot on trial and error when adjusting values like position and scale until things looked right. Iteration played a huge role in my development process because I was constantly making small improvements to the scene, whether it was fixing object placement or adjusting lighting to make it look better.
  Over time, my approach to coding has improved. At the beginning of the course, I relied more on following instructions exactly, but as I worked through the milestones, I became more comfortable experimenting and making changes on my own. I started to understand why certain things worked instead of just copying code, which made debugging easier and gave me more confidence overall.
  Computer science is helping me build problem-solving skills that I can use in my future career. It is also helping me get more comfortable learning new technologies, which is important in any tech-related job. This course specifically gave me a better understanding of how graphics and rendering work behind the scenes. These skills can be useful in future classes, especially ones related to programming or visualization.
  Even if I don’t directly work in graphics, the skills I learned in this project will still carry over into my professional path. Understanding how to break down problems, test solutions, and improve through iteration are all valuable in real-world development. This project also gave me something visual to include in my portfolio, which can help demonstrate my abilities to future employers.
